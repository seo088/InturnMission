1. 문서 개요

본 명세는 MCP Server·에이전트·스킬의 메타데이터를 시맨틱 웹 어휘로 표현하기 위한 두 어휘(mcpm:, am:)를 정의한다. v0.5는 HKUDS의 OpenHarness(2026-04-01) 등장에 따라 에이전트 인지 주체와 하네스 실행 인프라를 분리하는 구조 변경을 포함한다.

설계 원칙: LOV 재사용 우선. 기존 국제 표준 어휘로 표현 가능한 개념은 mcpm:/am:에 정의하지 않는다. MCP 프로토콜·Agent Harness에 고유한 개념만 신규 정의한다.

6. AIRD 연계 확장 속성 — 세 속성의 결합 효과

설계 의도: mcpm:hasScenario + am:executionTrace + am:governanceMode 세 속성이 결합되어야 STD-004 L3 인증의 자동화 쿼리가 완전해진다.
그러나 그것만이 아니다. 이 세 속성은 AIRD Pack의 소비 가능성 판정·품질 등급 자동 매핑·소비 Provenance 완성이라는 세 가지 독립적 효과를 동시에 가진다.

6-1. 세 속성의 단독 효과

속성단독 효과 (AIRD Pack)단독 효과 (STD-004)없으면 발생하는 문제
mcpm:hasScenario
+ am:feasibility
+ aird:qualityGate
+ am:triggerType		 소비 가능성 사전 판정.
AIRD Pack을 요청하기 전, Agent가 해당 Pack이 자신의 시나리오(ML/RAG/KG/Stats)에 적합한지와 최소 Q-Tier 요건을 온톨로지 쿼리로 확인. 부적합 Pack 요청 차단 → 파이프라인 낭비 제거.		 심사관이 서류로만 "이 Tool이 AIRD Pack을 소비할 수 있다"를 확인하는 대신, SPARQL 쿼리 1회로 Feasible 여부를 기계 판정. Agent가 Q-Bronze Pack을 Q-Gold 요구 ML 파이프라인에 투입 → 학습 품질 저하. 사후 발견 시 재처리 비용 발생.
am:executionTrace
→ prov:Activity
(PreToolUse~PostToolUse)		 소비 측 Provenance 완성.
AIRD가 정의하는 공급 측 Provenance(Raw→Gate→Pack)의 체인을 Tool 실행 이후까지 연장. "이 ML 모델은 어떤 AIRD Pack을 소비해 학습되었는가"를 역추적 가능.		 STD-004가 요구하는 "완전한 데이터 계보"를 달성. 공급 측만 검증하는 기존 ISO 8000 방식과의 차별화 근거. AIRD Pack이 AI Ready Data라는 인증을 받아도, 그것이 실제로 어떻게 소비되었는지 추적 불가 → "AI Ready"라는 주장이 공급 측 선언에 그침.
am:governanceMode
Default·Auto·Plan
+ am:hookType		 소비 환경 품질 보증.
AIRD Pack 소비 시 Harness의 권한 모드를 메타데이터로 기록. "이 Pack은 Auto 모드(완전 자율) 환경에서만 소비되었다"는 맥락 정보가 Pack의 신뢰도 평가에 반영.		 Governance 심사 항목(권한 제어·거부 규칙 존재 여부)을 SPARQL로 자동 판정. 현장 감사 없이 서류 대체. 동일한 AIRD Pack이 Default(수동 승인)와 Auto(완전 자율) 환경에서 소비된 결과를 구분 불가 → 재현 조건 불명확.

6-2. 결합 효과 매트릭스

세 속성은 단독으로도 효과가 있지만, 결합 시 각각의 합 이상의 효과를 만들어낸다.

결합AIRD Pack 효과STD-004 효과
hasScenario
+executionTrace		 소비 사전 적합성 → 사후 이력 연결.
"이 시나리오(RAG)에 적합하다고 판단한 Pack이 실제로 그 용도로 실행되었는가"를 end-to-end로 검증 가능.		 STD-004 파이프라인 적합성 심사 항목 ②③ 자동화.
서류 제출 없이 쿼리로 대체.
hasScenario
+governanceMode		 시나리오별 허용 거버넌스 매핑.
예: KG Pack은 GovernanceModeAuto만 허용, 의료 ML Pack은 GovernanceModeDefault(수동 승인) 필수. Pack 유형에 따른 소비 환경 정책을 온톨로지로 표현.		 STD-004 도메인 인증 트랙(의료·금융)별 거버넌스 요건을 선언적으로 기술 가능.
executionTrace
+governanceMode		 재현 조건 완전 명세.
동일 실행 이력(executionTrace)에 동일 거버넌스 모드가 결합되어야 진정한 재현성이 보장됨. GovernanceMode가 달라지면 동일 Pack으로도 다른 결과가 나올 수 있음을 명시.		 L3 현장 감사의 "기술 재현 테스트" 자동화.
심사관이 재현 조건(Pack URI + governanceMode)을 지정하면 SPARQL로 동일 조건의 기존 executionTrace와 비교.
세 속성 동시 결합
hasScenario
+executionTrace
+governanceMode		 완전한 AIRD Pack 소비 생애주기 명세.
소비 전(적합성 판정) → 소비 중(거버넌스 조건) → 소비 후(이력 기록)가 하나의 RDF 그래프로 연결. "AI Ready Data"의 AI 파이프라인 투입 가능성이 공급 측 선언이 아닌 소비 측 실행 이력으로 증명됨.		 STD-004 L3 완전 자동화 달성.
SPARQL 1 쿼리로:
① Pack의 Q-Tier 충족 여부
② Feasible 시나리오 매핑 존재
③ 실제 실행 이력 존재
④ Governance 조건 준수
→ 4개 심사 항목 동시 판정.

6-4. am:GovernanceMode 열거 인스턴스

인스턴스 URI레이블AIRD Pack 소비 맥락STD-004 연계
am:GovernanceModeDefaultDefault위험 작업 시 사용자 승인 필요. 의료·금융 도메인 ML Pack 필수.L3 도메인 인증 트랙(의료·금융)의 필수 거버넌스 조건
am:GovernanceModeAutoAuto신뢰 환경에서 완전 자율 실행. ML 배치 학습·KG 자동 갱신 허용.L2·L3 일반 인증 경로. executionTrace 자동 생성.
am:GovernanceModePlanPlan실행 없이 계획만 수립. Stats Pack의 집계 계획 수립 단계.L1 자가선언 단계에서 활용. 실행 이력 없으므로 executionTrace 미생성.

6-5. am:HookType 열거 인스턴스

인스턴스 URI레이블AIRD Pack 소비 효과STD-004 연계
am:HookPreToolUsePreToolUsePack 소비 Tool 호출 직전 — aird:qualityGate 충족 여부 자동 검사. 미충족 시 실행 차단 → 불량 Pack 투입 방지.Provenance 체인 시작점. prov:startedAtTime 기록 → STD-004 재현성 검증의 타임스탬프 시작.
am:HookPostToolUsePostToolUsePack 소비 Tool 실행 완료 후 — 결과 검증·executionTrace 완료 기록. 소비 측 Provenance 체인 종점 확정.Provenance 체인 종점. prov:endedAtTime 기록 → STD-004 L3 재현 테스트의 완료 조건.

7. SPARQL 활용 예제

# ① STD-004 L3 자동 검증: OpenHarness Auto 모드로 실행된 AIRD Pack 소비 이력
SELECT ?pack ?harness ?govMode ?trace ?qTier WHERE {
    ?agent am:runsOnHarness   ?harness ;
           am:executionTrace  ?trace .
    ?harness am:governanceMode am:GovernanceModeAuto ;
             am:mcpCompatible  true ;
             am:governanceMode ?govMode .
    ?tool  mcpm:connectedDataset ?pack ;
           prov:wasUsedBy       ?trace .
    ?pack  aird:qualityGate     ?qTier .
}

# ② Harness 비교: MCP 호환 + Skill 호환 하네스 목록 (OpenHarness vs 기타)
SELECT ?h ?title ?lang ?toolCnt ?swarm WHERE {
    ?h a am:AgentHarness ;
       dct:title              ?title ;
       am:harnessLang         ?lang ;
       am:toolCount           ?toolCnt ;
       am:swarmEnabled        ?swarm ;
       am:mcpCompatible       true ;
       am:skillsCompatible    true .
}
ORDER BY DESC(?toolCnt)

# ③ Skill별 AIRD Pack 소비 가능 여부 + 최소 Q-Tier 조회
SELECT ?skill ?title ?pack ?minQTier WHERE {
    ?skill a am:Skill ;
           dct:title        ?title ;
           am:usesMCPServer ?server ;
           mcpm:hasScenario [
               am:feasibility   am:Feasible ;
               aird:qualityGate ?minQTier
           ] .
    ?tool  mcpm:connectedDataset ?pack .
    ?server mcpm:tool ?tool .
}

# ④ OpenHarness 미사용 Agent의 STD-004 인증 위험 탐지
SELECT ?agent ?model WHERE {
    ?agent a am:Agent ;
           am:model ?model .
    FILTER NOT EXISTS {
        ?agent am:runsOnHarness ?h .
        ?h     am:governanceMode ?m .
        FILTER(?m != am:GovernanceModeDefault)
    }
}
