# 🐾 반려동물 지식 그래프 기반 웹 서비스 고도화 (Advanced Pet Service)

## 1. 연구 배경 (Background)
기존 서울시 중심의 데이터를 **전국 단위**로 확장하고, 사용자에게 신뢰성 있는 정보를 제공하기 위해 **데이터의 출처(Provenance)와 최신성(Recency)**을 명확히 보장하는 시스템을 구축합니다.

---

## 2. 연구 질문 (Research Questions, RQ) 

* **RQ 1 (Data Scalability):** 서울시에 한정된 데이터를 어떻게 **전국 단위**의 이기종 데이터(보호소, 문화시설 등)로 확장하여 통합할 것인가?
* **RQ 2 (Data Provenance & Reliability):** 실시간 API와 **정적 파일(Static Data)**을 혼용할 때, 사용자가 데이터의 **기준 시점(Timestamp)**과 **출처(Source)**를 명확히 인지할 수 있도록 메타데이터를 어떻게 관리할 것인가?
* **RQ 3 (Data Enrichment):** 공공데이터의 행정 정보(시설명, 위치)에 자체 구축한 **수의학 지식 그래프**를 융합하여 단순 조회 이상의 가치를 어떻게 창출할 것인가?

---

## 3. 향후 추진 계획 (Future Plans)

### 3.1. 데이터 수집 전략: 하이브리드 방식 (Hybrid Data Collection)
* **실시간 데이터 (API):** 변동성이 큰 데이터(유기동물 발생 현황, 입양 상태)는 **실시간 API**를 우선 활용.
* **정적 데이터 (File):** 변동이 적거나 API가 없는 전국 시설 데이터(카페, 병원 등)는 **최신 공공데이터 파일(CSV/JSON)**을 활용하되, **데이터 갱신 주기**를 시스템에 명시.

### 3.2. 메타데이터 명시 및 관리 (Metadata Visualization)
* **목표:** 교수님 피드백 반영 (데이터 신뢰성 확보)
* **구현 방안:**
    * 모든 데이터 카드 하단에 `Source`, `Last Updated`, `Reference Date` 필드 표기.
    * *UI 예시: "데이터 기준일: 2023.12.31 | 출처: 공공데이터포털"*
    * 정적 파일 업로드 시, 해당 파일의 메타 정보를 DB에 함께 적재하여 관리.

### 3.3. 전국 단위 데이터 매핑 (Standardization)
* 전국 공공데이터 표준 포맷("카테고리1", "시도 명칭" 등)을 준수하여, 지식 그래프(RDF) 스키마와 1:1 매핑 수행.
* 주소 정규화(Road Address Normalization)를 통해 전국 위치 기반 검색 기능 구현.

---

## 4. 데이터 명세 (Data Specification)

### 4.1. 전국 반려동물 동반 문화시설 (Static Data)
* **출처:** 공공데이터포털 (전국반려동물동반문화시설표준데이터)
* **형식:** JSON / CSV
* **주요 필드 매핑:**
    * `시설명` -> `schema:name`
    * `카테고리1/2/3` -> `schema:category`
    * `시도 명칭` + `도로명 이름` -> `schema:address`
    * `위도`, `경도` -> `schema:geo`
